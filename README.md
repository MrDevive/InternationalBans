# InternationalBans

[![Лицензия](https://img.shields.io/badge/License-Custom-red.svg)](LICENCE.md)
[![Java](https://img.shields.io/badge/Java-17-orange.svg)](https://adoptium.net/)

Плагин для централизованного бана игроков на Minecraft сервере с поддержкой MySQL и YML.

>
> Данный плагин защищён пользовательской лицензией. Вы **можете** собирать и использовать его на своих серверах, но вам **ЗАПРЕЩАЕТСЯ**:
> * Выкладывать файл плагина (`.jar`) или исходный код на сторонних сайтах, форумах, в Discord и т.д.
> * Продавать, перепродавать или распространять плагин любым другим способом.
>
> Полный текст лицензии с подробностями доступен в файле [`LICENCE.md`](LICENCE.md).

## 📋 Возможности

- **Централизованная система банов** — баны синхронизируются между несколькими серверами через MySQL
- **Поддержка временных банов** — возможность указать срок действия бана
- **Автоматическая синхронизация** — при бане на одном сервере, игрок блокируется на всех подключённых
- **Причины бана** — гибкая настройка причин и их отображения
- **История банов** — хранение всей истории наказаний игрока
- **Команды для модерации** — `/ban`, `/tempban`, `/unban`, `/warn`, `/history`, `/ib` и т.д

## 🔧 Требования

- Minecraft сервер версии **1.21+**
- **Java 17** или выше (для запуска сервера)
- **JDK 17** или выше (для сборки плагина из исходников)
- **MySQL сервер** (5.7+ или 8.0+)
- Права на создание таблиц в базе данных MySQL (пользователь должен иметь права `CREATE`, `INSERT`, `SELECT`, `UPDATE`, `DELETE`)

## 🛠️ Сборка плагина

После того как проект открыт и настроен, вы можете собрать его в работающий `.jar` файл.

**Через Maven (рекомендуется):**
1. Откройте панель `Maven` (обычно справа, значок `m`)
2. Раскройте свой проект → `Lifecycle`
3. Дважды кликните на `package`
4. Готовый `.jar` файл вы найдете в папке `target/`

**Через IDEA:**
1. Нажмите `Build` → `Build Artifacts...`
2. Выберите `InternationalBans:jar` → `Build`

## 📥 Установка на сервер

1. **Поместите файл** `.jar` из папки `target/` в папку `plugins/` вашего сервера Minecraft
2. **Запустите (или перезапустите) сервер** — плагин создаст папку `InternationalBans/` и файл `config.yml`
3. **Настройте `config.yml`** — укажите параметры подключения к MySQL (см. раздел [Конфигурация](#-конфигурация))
4. **Перезагрузите плагин** командой `/bans reload` или перезапустите сервер
5. **Повторите установку на каждом сервере**, где нужна синхронизация банов (используйте одну базу данных)

## ⚙️ Конфигурация

Плагин создаёт файл `config.yml` в папке `plugins/InternationalBans/`:

```yaml
settings:
  max-warnings: 3
  auto-ban-after-warnings: true
  auto-ban-duration: -1

  auto-ban-reason: "Превышение лимита предупреждений"

ban:
  save-ip-on-ban: true
  ban-all-accounts: true

ip-ban:
  ban-all-accounts-message: "&cВаш IP-адрес был заблокирован"

history:
  max-entries: 50
  sort-order: "newest_first"

database:
  type: yaml
  auto-save-interval: 6000

command-settings:
  require-confirm-reload: false
  reload-cooldown: 5
```

Файл `database.yml`:

```yaml
mysql:
  enabled: true

  host: "127.0.0.1"
  port: 3306
  database: "database_name"
  username: "root"
  password: "password"

  use-ssl: false
  allow-public-key-retrieval: true
  
  pool:
    max-pool-size: 10
    min-idle: 2
    connection-timeout: 30000
    idle-timeout: 600000
    max-lifetime: 1800000

  table-prefix: "international_bans"
```

## 🛠️ Команды и права

| Команда | Описание | Право (`permission`) |
| :--- | :--- | :--- |
| `/ban <игрок> [время] <причина>` | Забанить игрока навсегда (или с указанием времени) | `internationalbans.ban` |
| `/tempban <игрок> <время> <причина>` | Забанить игрока на время | `internationalbans.tempban` |
| `/unban <игрок>` | Снять бан с игрока | `internationalbans.unban` |
| `/ipban <игрок> <причина>` | Забанить игрока по IP навсегда | `internationalbans.ipban` |
| `/tempipban <игрок> <время> <причина>` | Забанить игрока по IP на время | `internationalbans.tempipban` |
| `/unbanip <игрок>` | Снять IP-бан с игрока | `internationalbans.unbanip` |
| `/mute <игрок> <причина>` | Замутить игрока навсегда | `internationalbans.mute` |
| `/tempmute <игрок> <время> <причина>` | Замутить игрока на время | `internationalbans.tempmute` |
| `/unmute <игрок>` | Снять мут с игрока | `internationalbans.unmute` |
| `/kick <игрок> <причина>` | Выгнать игрока с сервера | `internationalbans.kick` |
| `/warn <игрок> <причина>` | Выдать предупреждение игроку | `internationalbans.warn` |
| `/history <игрок>` | Показать историю наказаний игрока | `internationalbans.history` |
| `/reload` | Перезагрузить плагин | `internationalbans.reload` |
| `/ib help/info/stats` | Главная команда плагина (помощь, информация, статистика) | `internationalbans.admin` |

> **📌 Примечание:** У команд есть альтернативные названия (алиасы):
> - `/ban` → `/permban`
> - `/tempban` → `/tban`
> - `/unban` → `/pardon`
> - `/tempipban` → `/tipban`
> - `/unbanip` → `/pardonip`
> - `/tempmute` → `/tmute`
> - `/history` → `/punishments` или `/bans`
> - `/reload` → `/ibrl`

**Дополнительные права:**
- `internationalbans.bypass` — обход наказаний (могут игнорировать баны/муты)
- `internationalbans.reload.bypass` — обход кулдауна перезагрузки
- `internationalbans.notify` — получение уведомлений о наказаниях (например, при бане/муте на сервере)

## ❓ Часто задаваемые вопросы (FAQ)

**Вопрос:** Можно ли использовать плагин на коммерческом сервере?

**Ответ:** Да. Вы можете использовать плагин на любом сервере, включая коммерческие проекты.

**Вопрос:** Могу ли я изменить код плагина под себя?

**Ответ:** Да, для личного использования на своём сервере. Но вы не имеете права распространять модифицированную версию без письменного разрешения автора.

**Вопрос:** Как синхронизировать баны между несколькими серверами?

**Ответ:** Установите плагин на каждый сервер и настройте `config.yml` с одними и теми же параметрами MySQL на всех серверах. Баны будут автоматически синхронизироваться.

**Вопрос:** Плагин не видит базу данных. Что делать?

**Ответ:** Проверьте параметры в config.yml, убедитесь, что MySQL сервер запущен и доступен по указанному хосту и порту. Также проверьте, что пользователь имеет права на создание таблиц.

**Вопрос:** Поддерживаются ли имена игроков с русскими буквами?

**Ответ:** Да, плагин корректно работает с UUID игроков, поэтому любые имена поддерживаются.

**Вопрос:** Что происходит с игроком, если его забанили на одном сервере?

**Ответ:** Если настроена синхронизация, при попытке зайти на любой другой сервер из сети, игрок получит сообщение о бане и не сможет зайти.
